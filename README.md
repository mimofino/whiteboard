# Distributed Shared Whiteboard

A multi-user real-time collaborative whiteboard built with Java RMI. Multiple users can draw shapes, annotate with text, and chat on a shared canvas simultaneously. One user is the **manager** (session owner); all others join as **peers**.

---

## Requirements

- Java 8 or later
- No external dependencies

---

## Build

```bash
bash compile.sh
```

Compiled `.class` files are placed in the `out/` directory.

---

## Run

Open **three separate terminals**, all from the `whiteboard/` directory.

### 1. Start the server
```bash
java -cp out server.WhiteboardServer <port>
```
Example:
```bash
java -cp out server.WhiteboardServer 8888
```

### 2. Launch the manager
```bash
java -cp out client.CreateWhiteBoard <serverIP> <port> <username>
```
Example:
```bash
java -cp out client.CreateWhiteBoard localhost 8888 Alice
```

### 3. Join as a peer
```bash
java -cp out client.JoinWhiteBoard <serverIP> <port> <username>
```
Example:
```bash
java -cp out client.JoinWhiteBoard localhost 8888 Bob
```

> The peer will wait for the manager to approve the join request. A confirmation dialog appears on the manager's screen.

## Features

### Drawing Tools
| Tool | Description |
|------|-------------|
| Line | Click and drag to draw a straight line |
| Rectangle | Click and drag to draw a rectangle |
| Circle | Click and drag to draw an ellipse |
| Triangle | Click and drag; apex at midpoint, base at bottom |
| Free Draw | Freehand stroke following the mouse |
| Text | Click to place text; choose font size in dialog |
| Eraser | Paints over shapes in white (S / M / L sizes) |

- **Stroke width**: 1 / 2 / 3 / 5 / 8 px
- **Colour picker**: 6×10 grid (grayscale row + 5 hue/shade rows)

### Collaboration
- All drawing operations are propagated to peers in real time via RMI callbacks
- Built-in chat panel for text messaging between all participants
- Live online-user list, updated whenever anyone joins or leaves

### Manager Privileges
| Action | Description |
|--------|-------------|
| Approve / Reject join | Shown as a confirm dialog when a peer requests to join |
| Kick user | Select a user in the list and click "Kick Selected User" |
| New | Clears the canvas for all users |
| Open | Loads a `.wb` file and syncs the canvas to all users |
| Save / Save As | Saves the current canvas to a `.wb` binary file |
| Close | Terminates the session; all peers are notified and exit |

Peers can draw and chat but cannot perform any of the above operations.

---

## File Format

Canvases are saved as `.wb` files using Java object serialisation (`ObjectOutputStream`). Files can only be opened by this application.

## Architecture Overview

```
┌─────────────────────────────────────────┐
│              WhiteboardServer           │
│         (WhiteboardServiceImpl)         │
│  - state: WhiteboardState               │
│  - activeUsers: LinkedHashMap           │
│  - pendingUsers: LinkedHashMap          │
└──────────────┬──────────────────────────┘
               │ Java RMI (bidirectional)
       ┌───────┴────────┐
       │                │
┌──────┴──────┐  ┌──────┴──────┐
│  Manager    │  │    Peer     │
│  Client     │  │   Client    │
│ (Creator)   │  │  (Joiner)   │
└─────────────┘  └─────────────┘
```

- **Client → Server**: `WhiteboardService` (14 methods) — drawing, chat, join, kick, close
- **Server → Client**: `ClientCallback` (11 methods) — push events for shapes, chat, user list, session state

---

## Notes

- Username must be unique across both active and pending users
- If a client crashes, the server evicts it lazily on the next failed RMI call
- The eraser appends a white `FreeDrawShape` to the shape list; it does not delete existing shapes
- Replace `localhost` with the server machine's LAN IP for multi-machine demos
